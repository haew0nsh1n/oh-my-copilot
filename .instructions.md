# oh-my-copilot Development Instructions

You are helping develop **oh-my-copilot**, a Python agent framework with engineering skills.

## Core Principles

1. **Test-First** - Write tests before implementation
2. **Domain-Driven** - Domain model is the source of truth
3. **Deep Modules** - Complex internals, simple interfaces
4. **Skill-Based** - Reusable engineering workflows

## Key Guidelines

### When Implementing Features
- [ ] Write failing test first (RED)
- [ ] Implement to pass test (GREEN)
- [ ] Refactor if needed (REFACTOR)
- [ ] Update domain model if needed
- [ ] Ensure 85%+ test coverage

### When Fixing Bugs
- [ ] Create test that reproduces bug
- [ ] Fix the bug
- [ ] Ensure related tests still pass
- [ ] Document root cause in comments

### Code Quality
- Type hints on all functions
- Docstrings on all public APIs
- Error messages must be clear
- Follow existing code style

### Testing
- Use pytest fixtures for common setup
- Test classes group related tests
- Test names describe behavior (test_X_can_Y)
- Mock external dependencies

## Current Workspace Skill Layer

Use the local `.agents/skills/` directory as the single agent-facing skill layer for this repository. Treat `src/skills/` and the `oh-my-copilot` CLI as the product being developed, not as the layer that defines which assistant skills are available in this workspace.

Available local engineering skills include:

- `/codebase-design` - Design module interfaces
- `/design-an-interface` - Compare API/interface shapes
- `/diagnosing-bugs` - Debug systematically
- `/domain-modeling` - Define ubiquitous language
- `/grilling` - Stress-test plans and designs
- `/request-refactor-plan` - Plan refactorings
- `/review` - Code review (standards + spec)
- `/tdd` - Test-driven development workflow

- [ ] All tests pass: `pytest`
- [ ] No type errors: `mypy src/`

- [ ] Linter clean: `ruff check src/ tests/`
